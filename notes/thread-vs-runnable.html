Thread 子类 vs Runnable 接口创建线程的区别

2026-06-24

Java 进阶

引言：两种创建线程的方式

学习 Java 多线程时，你一定见过两种创建线程的写法：继承 Thread 类 和 实现 Runnable 接口。 很多初学者会觉得"效果一样，随便用哪个都行"，但实际上它们在设计思想、扩展性、资源共享等方面有本质区别。 本文将从实际代码出发，帮你彻底搞清楚两者的差异，以及什么时候该用哪个。

一、两种方式的基本写法

方式一：继承 Thread 类

通过创建 Thread 的子类，重写 run() 方法来定义线程要执行的任务：

/**
 * 方式一：继承 Thread 类
 */
class MyThread extends Thread {
    private String taskName;

    public MyThread(String taskName) {
        this.taskName = taskName;
    }

    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(taskName + " 执行第 " + i + " 次 —— 线程：" + getName());
        }
    }
}

public class ThreadDemo {
    public static void main(String[] args) {
        MyThread t1 = new MyThread("任务A");
        MyThread t2 = new MyThread("任务B");
        t1.start();
        t2.start();
    }
}

方式二：实现 Runnable 接口

定义一个实现 Runnable 接口的类，然后把它传给 Thread 的构造方法：

/**
 * 方式二：实现 Runnable 接口
 */
class MyRunnable implements Runnable {
    private String taskName;

    public MyRunnable(String taskName) {
        this.taskName = taskName;
    }

    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(taskName + " 执行第 " + i + " 次 —— 线程：" + Thread.currentThread().getName());
        }
    }
}

public class RunnableDemo {
    public static void main(String[] args) {
        Thread t1 = new Thread(new MyRunnable("任务A"), "线程-1");
        Thread t2 = new Thread(new MyRunnable("任务B"), "线程-2");
        t1.start();
        t2.start();
    }
}

两种方式都能正常创建线程、并发执行任务。那区别到底在哪？

二、核心区别对比

对比维度 继承 Thread 类 实现 Runnable 接口
Java 继承限制 只能单继承，占用了唯一的继承名额 可以同时实现多个接口，灵活无限制
资源共享 每个子类实例有独立的成员变量，共享资源不方便 同一个 Runnable 实例可传给多个 Thread，天然支持资源共享
职责分离 任务逻辑和线程机制耦合在一个类中 任务（Runnable）和线程（Thread）解耦，符合单一职责
线程池兼容 不能直接提交给 ExecutorService 可以直接提交给线程池，天然兼容
扩展性 继承后无法再继承其他类 实现 Runnable 后仍可继承其他类

三、区别一：Java 单继承的限制

这是最根本的区别。Java 只支持单继承，如果你的类已经继承了 Thread，就无法再继承其他任何类。 

// 继承 Thread 的问题：
class MyThread extends Thread {  // 已经继承了 Thread
    // 如果我还想继承别的类？不行！
    // class MyThread extends SomeOtherClass ← 编译错误！
}

// 实现 Runnable 就没这个问题：
class MyTask implements Runnable {  // 实现接口，不影响继承
    // 仍然可以继承其他类
}

class MyTask extends SomeBaseClass implements Runnable {  // ✅ 完全合法
    @Override
    public void run() {
        // 线程任务逻辑
    }
}

在实际项目中，你的业务类往往需要继承某个框架的基类（比如 Servlet、Activity 等）， 这时候用 Runnable 接口是唯一的选择。

四、区别二：资源共享能力

这是多线程编程中最常遇到的实际问题。假设我们有一个卖票系统，多个窗口同时卖票：

用 Thread 子类（每个线程各自持有票数，无法共享）：

// ⚠️ 有问题的写法：每个线程有自己的 tickets，各卖各的
class TicketThread extends Thread {
    private int tickets = 10;   // 每个实例各持有 10 张票

    @Override
    public void run() {
        while (tickets > 0) {
            System.out.println(getName() + " 卖出第 " + tickets-- + " 张票");
        }
    }
}

public class BadTicketDemo {
    public static void main(String[] args) {
        // 三个窗口各卖各的 10 张，总共卖了 30 张——不是我们要的效果！
        new TicketThread().start();
        new TicketThread().start();
        new TicketThread().start();
    }
}

用 Runnable 接口（多个线程共享同一个 Runnable 实例）：

// ✅ 正确的写法：多个线程共享同一个 Runnable 对象的票数
class TicketRunnable implements Runnable {
    private int tickets = 10;   // 只有一份票数，被所有线程共享

    @Override
    public void run() {
        while (true) {
            if (tickets <= 0) break;
            System.out.println(Thread.currentThread().getName() + " 卖出第 " + tickets-- + " 张票");
        }
    }
}

public class GoodTicketDemo {
    public static void main(String[] args) {
        Runnable task = new TicketRunnable();  // 创建一个任务对象

        // 三个线程共享同一个 task，共同操作同一份 tickets
        new Thread(task, "窗口-1").start();
        new Thread(task, "窗口-2").start();
        new Thread(task, "窗口-3").start();
    }
}

关键点：Runnable task = new TicketRunnable() 只创建了一个对象， 然后把这个对象传给三个 Thread。三个线程执行的是同一个 task.run()， 访问的是同一个 tickets 变量，这才是真正的资源共享。

五、区别三：职责分离的设计思想

从面向对象设计的角度看，Runnable 体现的是"组合优于继承"的设计原则：

  • Thread：负责线程的创建、调度、生命周期管理（线程机制）
  • Runnable：负责定义具体的任务逻辑（业务逻辑）
// 继承方式：一个类既管"怎么做线程"又管"做什么任务"——职责混合
class MyThread extends Thread {
    @Override
    public void run() {
        // 业务逻辑 和 线程机制 耦合在一起
    }
}

// 接口方式：各司其职
class DownloadTask implements Runnable {       // 只关心"做什么"——下载文件
    @Override
    public void run() {
        downloadFile(url);
    }
}

// Thread 只关心"怎么做"——创建线程、启动、调度
Thread t = new Thread(new DownloadTask(), "下载线程");
t.start();

这种分离让代码更灵活：同一个 DownloadTask 可以交给 Thread 执行， 也可以交给线程池执行，还可以在不需要线程时直接调用 run() 做单元测试。

六、区别四：线程池兼容性

在实际开发中，我们几乎不会直接 new Thread，而是使用线程池（ExecutorService）来管理线程。 线程池的 submit() 和 execute() 方法接受的是 Runnable 或 Callable，而不是 Thread 子类： 

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadPoolDemo {
    public static void main(String[] args) {
        ExecutorService pool = Executors.newFixedThreadPool(3);

        // ✅ Runnable 可以直接提交给线程池
        pool.submit(new Runnable() {
            @Override
            public void run() {
                System.out.println("任务1：" + Thread.currentThread().getName());
            }
        });

        // ✅ Lambda 更简洁（Runnable 是函数式接口）
        pool.submit(() -> {
            System.out.println("任务2：" + Thread.currentThread().getName());
        });

        // ❌ Thread 子类不能直接提交（它不是 Runnable 的直接实现）
        // pool.submit(new MyThread());  // 虽然 Thread 实现了 Runnable，但这是滥用

        pool.shutdown();
    }
}

虽然 Thread 类本身也实现了 Runnable 接口，但把 Thread 对象提交给线程池是反模式—— 线程池自己管理线程的创建和销毁，你提交的应该只是"任务"，而不是"线程"。

七、实际开发中的最佳实践

记住一条黄金法则：

优先使用 Runnable 接口（或 Lambda），不要继承 Thread。

理由汇总：

  1. 不占用单继承名额，保留类的扩展能力
  2. 天然支持资源共享（多个线程执行同一个 Runnable）
  3. 任务逻辑和线程机制解耦，代码更清晰
  4. 兼容线程池（ExecutorService），这是生产环境的标准用法
  5. Java 8 之后可以用 Lambda 简化，写起来比继承更简洁
// 现代 Java 推荐写法：Lambda + 线程池
ExecutorService pool = Executors.newFixedThreadPool(4);

pool.submit(() -> {
    System.out.println("简洁又优雅");
});

pool.shutdown();

八、常见疑问：Thread 子类就一无是处吗？

也不是。在以下少数场景中，继承 Thread 也有其合理性：

  • 需要重写 Thread 的其他方法：比如自定义 interrupt() 行为、修改线程名称生成逻辑等。
  • 线程本身需要携带复杂状态：线程对象本身就是业务实体（比如自定义的 Worker 线程，有自己的生命周期管理方法）。
  • 学习和理解原理：继承 Thread 是理解多线程的第一步，有助于掌握线程的本质。
// 合理使用 Thread 子类的例子：自定义线程行为
class DaemonWorker extends Thread {
    private volatile boolean running = true;

    public DaemonWorker(String name) {
        super(name);
        setDaemon(true);   // 设为守护线程
    }

    public void stopWorker() {
        running = false;
    }

    @Override
    public void run() {
        while (running) {
            System.out.println(getName() + " 正在执行后台任务...");
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                break;
            }
        }
        System.out.println(getName() + " 已停止");
    }
}

总结

一句话总结两种方式的本质区别：

  • 继承 Thread："我就是线程" —— 任务逻辑和线程身份绑死。
  • 实现 Runnable："我可以在线程里跑" —— 任务逻辑独立于线程，灵活组合。

在 99% 的实际开发场景中，Runnable（或 Lambda）+ 线程池 是正确答案。 继承 Thread 更适合学习阶段理解原理，以及少数需要深度定制线程行为的场景。 掌握这个区别，是写出高质量多线程代码的第一步。
