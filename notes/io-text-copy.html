Java IO 流之文本文件复制：字符缓冲流保姆级教程

2026-05-23

Java 基础

引言：为什么要学文件复制？

文件复制是编程中最基础、最常用的操作之一。无论是备份配置文件、导出日志、还是处理用户上传的文本，都绕不开 IO 流。 在 Java 中，处理文本文件最推荐的方案就是 BufferedReader + BufferedWriter 这对黄金组合。 本篇文章会用保姆级的方式，从零带你掌握文本文件复制。

一、基础概念：什么是 IO 流？

想象一根水管：水从源头（Source）流到目的地（Destination）。Java 的 IO 流就是这根"数据水管"—— 数据从数据源（文件、网络、内存）流向目标位置。

Java 的 IO 流按处理单位分为两大类：

类型处理单位典型类适用场景
字节流byte（字节）InputStream / OutputStream图片、视频、任意文件
字符流char（字符）Reader / Writer纯文本文件（.txt .java .html 等）

对于文本文件，字符流是天然的选择——因为它自带编码处理，能正确识别中文、英文、换行符等字符边界，不会出现半个汉字的乱码问题。

二、为什么需要缓冲流？

你可能会问：既然 FileReader 和 FileWriter 已经能读写文件了，为什么还要用 BufferedReader / BufferedWriter？

答案就一个字：慢。

没加缓冲的流，就像这样：

// ❌ 低效方式：每次只读一个字符，就要访问一次磁盘
FileReader reader = new FileReader("source.txt");
int ch;
while ((ch = reader.read()) != -1) {  // 读一个字符 → 访问磁盘 → 读一个字符 → 访问磁盘...
    // ...
}

每次 read() 调用都会触发一次磁盘 IO，而磁盘 IO 比内存操作慢几个数量级。这就好比去超市买东西，一次只拿一个商品就结账，反复跑无数次——效率极低。

加了缓冲之后：

// ✅ 高效方式：一次从磁盘读一大块（默认 8192 个字符），放在内存缓冲区里
BufferedReader reader = new BufferedReader(new FileReader("source.txt"));
String line;
while ((line = reader.readLine()) != null) {  // 从内存缓冲区取，不是每次访问磁盘
    // ...
}

缓冲流内部维护了一个内存缓冲区（默认 8192 字符/字节）。它一次从磁盘读一大块数据放入缓冲区，后续的读取操作直接从内存取， 直到缓冲区空了再去磁盘取下一块。这就是"少跑几趟、每次多运点"的策略。

缓冲流的核心优势总结：

对比维度无缓冲（FileReader/FileWriter）有缓冲（BufferedReader/BufferedWriter）
磁盘访问次数每次 read/write 都访问大幅减少，批量处理
速度慢快（通常 10~100 倍差距）
便捷方法只有单字符读写提供 readLine() / newLine()
资源开销一般多一点点内存（缓冲区）

三、核心方法速览

在动手写代码之前，先认识一下这对黄金组合的常用方法：

// BufferedReader 常用方法
String line = reader.readLine();    // 读取一行文本，读到末尾返回 null
reader.close();                     // 关闭流，释放资源

// BufferedWriter 常用方法
writer.write(String s);             // 写入字符串
writer.newLine();                   // 写入一个换行符（跨平台兼容！）
writer.flush();                     // 强制将缓冲区内容刷到磁盘
writer.close();                     // 关闭流（会自动 flush）

特别提醒：newLine() 是一个很贴心的方法。它会根据操作系统自动输出正确的换行符 （Windows 是 \r\n，Linux/Mac 是 \n），不用手动拼接。

四、代码实战：按行复制文本文件

下面是一个完整的、可以直接运行的文本文件复制示例：

import java.io.*;

/**
 * 文本文件复制 —— 使用 BufferedReader + BufferedWriter
 * 功能：将 source.txt 的内容按行复制到 target.txt
 */
public class TextFileCopy {
    public static void main(String[] args) {
        // 源文件路径和目标文件路径（请根据实际情况修改）
        String sourcePath = "source.txt";
        String targetPath = "target.txt";

        // 在 try() 中声明资源，Java 会自动调用 close()，不用写 finally
        try (
            // 第一层：FileReader —— 建立到源文件的字符连接
            FileReader fr = new FileReader(sourcePath);
            // 第二层：BufferedReader —— 套上缓冲"外衣"，提供 readLine() 等高效方法
            BufferedReader reader = new BufferedReader(fr);

            // 同理，写出端也是两层包装
            FileWriter fw = new FileWriter(targetPath);
            BufferedWriter writer = new BufferedWriter(fw);
        ) {
            String line;           // 用来暂存每一行的内容
            int lineCount = 0;     // 计数器：一共复制了多少行

            // readLine() 每次读取一行，读到文件末尾时返回 null
            while ((line = reader.readLine()) != null) {
                writer.write(line);      // 将当前行写入目标文件
                writer.newLine();        // 补上换行符（readLine() 不会返回换行符）
                lineCount++;
            }

            System.out.println("复制完成！共复制 " + lineCount + " 行。");

        } catch (FileNotFoundException e) {
            System.out.println("找不到文件：" + e.getMessage());
        } catch (IOException e) {
            System.out.println("读写出错：" + e.getMessage());
        }
    }
}

五、逐行解读：代码为什么这样写？

下面把关键点拆开来讲，帮你彻底理解每一行代码在干什么：

5.1 try-with-resources：自动关流

try (
    FileReader fr = new FileReader(sourcePath);
    BufferedReader reader = new BufferedReader(fr);
    // ...
) {
    // 使用流...
}  // ← 这里会自动调用 fr.close(), reader.close(), fw.close(), writer.close()

这是 Java 7 引入的语法，强烈建议使用。放在 try( ) 括号里的资源，在 try 代码块结束后 自动关闭，无论是否抛出异常。告别手写 finally { close() } 的繁琐。

回顾一下老写法有多啰嗦：

// 😫 Java 7 以前的写法（不推荐，仅为了对比）
BufferedReader reader = null;
BufferedWriter writer = null;
try {
    reader = new BufferedReader(new FileReader("source.txt"));
    writer = new BufferedWriter(new FileWriter("target.txt"));
    // ...
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (reader != null) { try { reader.close(); } catch (IOException e) { } }
    if (writer != null) { try { writer.close(); } catch (IOException e) { } }
}

5.2 装饰器模式：层层包装

BufferedReader reader = new BufferedReader(new FileReader("source.txt"));
//         ↑ 缓冲功能              ↑ 基础读取功能

这里是经典的装饰器模式（Decorator Pattern）：BufferedReader 不直接操作文件， 而是"包装"一个 FileReader，在它基础上增加缓冲功能。你可以把它理解为： FileReader 是汽车引擎，BufferedReader 是给这辆车装上了涡轮增压。

5.3 readLine() 的注意点

while ((line = reader.readLine()) != null) {
    writer.write(line);
    writer.newLine();  // ⚠️ 别忘了手动加换行！
}

readLine() 返回的字符串不包含换行符。如果你只写 write(line) 而不加 newLine()， 目标文件里所有的内容会挤在一行！这是一个初学者常踩的坑。

六、进阶技巧：指定字符编码

上面的例子使用了默认编码。如果你的文件是 UTF-8 编码，而系统默认是 GBK（常见于中文 Windows），就会出现乱码。 解决方法是使用 InputStreamReader 和 OutputStreamWriter 来指定编码： 

import java.io.*;
import java.nio.charset.StandardCharsets;

public class TextFileCopyWithEncoding {
    public static void main(String[] args) {
        String sourcePath = "source.txt";
        String targetPath = "target.txt";

        try (
            // InputStreamReader：字节流 → 字符流 的桥梁，可以指定编码
            InputStreamReader isr = new InputStreamReader(
                new FileInputStream(sourcePath), StandardCharsets.UTF_8);
            BufferedReader reader = new BufferedReader(isr);

            OutputStreamWriter osw = new OutputStreamWriter(
                new FileOutputStream(targetPath), StandardCharsets.UTF_8);
            BufferedWriter writer = new BufferedWriter(osw);
        ) {
            String line;
            while ((line = reader.readLine()) != null) {
                writer.write(line);
                writer.newLine();
            }
            System.out.println("复制完成（UTF-8 编码）！");
        } catch (IOException e) {
            System.out.println("操作失败：" + e.getMessage());
        }
    }
}

七、Buffer 大小可以调吗？

可以。BufferedReader 和 BufferedWriter 默认缓冲区是 8192 个字符（约 16KB）。 你也可以指定自定义大小： 

// 指定缓冲区大小为 32768 个字符（约 64KB）
BufferedReader reader = new BufferedReader(new FileReader("source.txt"), 32768);
BufferedWriter writer = new BufferedWriter(new FileWriter("target.txt"), 32768);

一般来说，默认的 8K 缓冲区对大多数场景已经足够了。除非你在处理超大文本文件且需要极致优化，否则不需要调整。

总结

回顾一下本文的核心知识点：

  • 字符流（Reader/Writer）专门处理文本文件，自带编码支持。
  • 缓冲流（BufferedReader/BufferedWriter）通过内存缓冲区大幅减少磁盘 IO 次数，是文本文件复制的最佳实践。
  • readLine() + write() + newLine() 是最常用的文本复制三板斧。
  • 用 try-with-resources 自动管理资源，简洁又安全。
  • 指定编码时，在中间加一层 InputStreamReader / OutputStreamWriter。

掌握了字符缓冲流，你就掌握了处理文本文件最优雅的方式。下一篇我们会讲字节缓冲流，看看如何复制图片、视频等"非文本"文件。
