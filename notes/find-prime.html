如何快速找出素数

2026-05-09

什么是素数

素数（质数）是指在大于 1 的自然数中，只能被 1 和它本身整除的数。 例如：2, 3, 5, 7, 11, 13, 17, 19 ...
如果一个数能被分解成两个更小的数的乘积，它就是合数。1 既不是素数也不是合数。

朴素方法（试除法）

最直观的思路：对每个数 n，尝试用 2 到 √n 之间的所有数去除它。

// 判断一个数是不是素数
public static boolean isPrime(int n) {
    if (n < 2) return false;
    for (int i = 2; i * i <= n; i++) {
        if (n % i == 0) return false;
    }
    return true;
}

// 找出 1 ~ n 范围内的所有素数
public static List<Integer> findPrimes(int n) {
    List<Integer> list = new ArrayList<>();
    for (int i = 2; i <= n; i++) {
        if (isPrime(i)) list.add(i);
    }
    return list;
}

这个方式在 n 比较小的时候没问题，但 n 一大就慢了。找 1~10 万 需要判断每个数，时间复杂度是 O(n√n)。

埃拉托斯特尼筛法（Sieve of Eratosthenes）

这是一种古老而优雅的算法，大约公元前 240 年由古希腊数学家提出。 核心思想只有一句话：从 2 开始，把每个素数的所有倍数都划掉，剩下的就是素数。

算法图解（以 n = 30 为例）：

  1. 列出 2 到 30 的所有数。
  2. 2 是素数 → 把 4,6,8,10…（2 的倍数）全部划掉。
  3. 下一个没被划掉的数是 3 → 它是素数 → 划掉 6,9,12…（3 的倍数）。
  4. 下一个没被划掉的数是 5 → 它是素数 → 划掉 10,15,20…（5 的倍数）。
  5. 重复，直到 √n = √30 ≈ 5.5，即处理完 5 之后就可以停了。

Java 实现

public static List<Integer> sieveOfEratosthenes(int n) {
    // isPrime[i] = true 表示 i 目前还没被划掉
    boolean[] isPrime = new boolean[n + 1];
    Arrays.fill(isPrime, true);
    isPrime[0] = isPrime[1] = false;  // 0 和 1 不是素数

    for (int i = 2; i * i <= n; i++) {
        if (isPrime[i]) {
            // 把 i 的所有倍数标记为合数
            // 从 i*i 开始，因为更小的倍数已经被处理过
            for (int j = i * i; j <= n; j += i) {
                isPrime[j] = false;
            }
        }
    }

    // 收集结果
    List<Integer> primes = new ArrayList<>();
    for (int i = 2; i <= n; i++) {
        if (isPrime[i]) primes.add(i);
    }
    return primes;
}

为什么筛法这么快

跟朴素方法对比一下：

方法时间复杂度找 100 万以内素数
试除法（朴素）O(n√n) ≈ 10⁹ 次运算数秒到十几秒
埃拉托斯特尼筛法O(n log log n) ≈ 2×10⁷ 次运算几十毫秒

筛法快的三个原因：

  1. 内层循环从 i² 开始：当 i=5 时，10, 15, 20 已经被 2 和 3 处理过了，直接从 25 开始即可。
  2. 只需遍历到 √n：任何合数 n 必然有一个因子 ≤ √n，所以外层循环到 √n 就足够了（i * i <= n）。
  3. bool 数组操作极快：标记一个位置为 false 只是 O(1) 的数组访问，CPU 缓存非常友好。

多亏了 log log n 这个因子的缓慢增长，即使 n 达到一亿，筛法也能在 1 秒内完成。

小结

埃拉托斯特尼筛法用「空间换时间」的思路，用 O(n) 的 bool 数组换来了 O(n log log n) 的高效筛选。 遇到需要批量找素数的场景，优先用筛法；只在判断单个数时，才用试除法。
